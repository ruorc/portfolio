
добавь комментарии в стиле Javadoc/Clean Code на английском



Рекомендации по стилю:
имнеа функций и переменных должны передавать их значение/назначение.
CamelCase для переменных, функций, методов и свойств объектов (navigateTo, currentPage).
PascalCase для имен модулей-синглтонов (экспортируемых объектов), таких как Router, UI, Api, Config.
SCREAMING_SNAKE_CASE только для истинных констант в файле constants.js (CACHE_TTL, PAGE_HOME).
Стрелочные функции для простых геттеров, обычные методы внутри объектов (для читаемости).
Префикс _ (underscore) для приватных методов, которые не должны вызываться снаружи модуля (например, в Loader).
«защитное» программированием (Default Parameters и Null Checks).
